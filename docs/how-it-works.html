ขั้นตอนการทำงาน

Motherflame ทำงานยังไง

1

🔌 เชื่อม AI + สร้าง Org Brain

motherflame setup ใส่ API key (Anthropic/OpenAI/Ollama) → motherflame create "ชื่อบริษัท" สร้าง brain + Flame Key (กุญแจประจำองค์กร)

↓
2

🌾 Harvest — เก็บความรู้เข้า brain (4 ทาง)

research <url> AI อ่านเว็บบริษัท · start สแกนไฟล์ (md/pdf/html) · chat คุยแล้วบอก fact · docs add เก็บแผนยาวทั้งฉบับ
ทุกข้อมูลถูกสกัดเป็น fact (สั้น) หรือ document (ยาว) — และ ยืนยันก่อนเข้า brain

↓
3

⚖️ Resolve — ตัดสินความจริงจากหลักฐาน

เก็บทุกคำพูดเป็น claims (ไม่ทับกัน) แล้วคำนวณ ความจริง ด้วย resolution ladder: คนตัดสินเอง › เจ้าของหมวด › เสียงส่วนใหญ่ › trust score

↓
4

🔍 Retrieve — ดึงความรู้ที่ถูกให้ AI

query/chat → จัดอันดับ fact + document chunk ที่ตรงคำถาม → ส่งเฉพาะที่สำคัญ (token budget) ให้ LLM ตอบ พร้อมที่มา

↓
5

🔐 Sync — แชร์ทั้งทีม (Zero-Knowledge)

push/pull เข้ารหัส brain ที่เครื่องด้วย AES-256-GCM ก่อนส่งขึ้น git repo → host เห็นแต่ ciphertext · เพื่อนร่วมทีม join ด้วย Flame Key เดียวกัน

↓
6

🤖 Connect — ให้ AI agent ทุกตัวเชื่อม

motherflame mcp เปิด MCP server → Claude Code / Cursor / agent ใดๆ query & เขียน brain ได้ — รู้บริบทบริษัทโดยไม่ต้องบอกซ้ำ

การเก็บข้อมูล

โครงสร้างข้อมูล — 2 ชั้น

ทุกอย่างเป็น JSON ในเครื่องที่ ~/.motherflame/ — ไม่มี server, ไม่มี database, ผู้ใช้เป็นเจ้าของข้อมูล 100%

แหล่งข้อมูล
ไฟล์ · เว็บ · แชต · แผน
→
CLAIMS
หลักฐานดิบทุกคำพูด
(ไม่เคยทับ/ลบ)
→
CANONICAL (items)
ความจริง 1 ค่า/key
คำนวณจาก claims
→

อ่านผ่าน MCP

💡 หัวใจของ design: แยก "หลักฐาน" (claims) ออกจาก "ความจริง" (items) — เพราะ knowledge base ทั่วไปใช้ "คนล่าสุดถูกเสมอ" ซึ่งทำลายข้อมูลเงียบๆ

config.json — ตั้งค่า + Flame Key brain.json — claims/items/documents ledger.json — ที่มาของทุก fact sessions/ — ประวัติแชต cloud/ — ciphertext (encrypted)

สิ่งที่ควรรู้ & ควรระวัง

✅ สิ่งที่ควรรู้

เป็นเจ้าของข้อมูลเอง
ทุกอย่างอยู่ในเครื่อง ไม่มี server กลาง — fork ไปใช้เองได้ (open-source)
Zero-knowledge sync
brain ถูก encrypt ที่เครื่องก่อนส่ง — GitHub/host เห็นแต่ ciphertext เท่านั้น
Bring-your-own AI key
ใช้ key ของคุณเอง (OpenAI/Claude/Ollama) — ไม่ผูกกับ service ใคร
ข้อมูลขัดแย้งถูกจับ
ถ้าทีมพูดไม่ตรงกัน ระบบ flag เป็น "contested" ไม่มั่วตอบว่าอันไหนจริง
ทำงานได้แม้ไม่มี key
keyword mode สกัด/ค้นได้โดยไม่ส่งข้อมูลออกเลย (offline)
ยืนยันก่อนเข้า brain
research/review ให้คน confirm fact ก่อนเสมอ — ไม่เก็บมั่ว

⚠️ สิ่งที่ควรระวัง

AI harvest ส่งข้อมูลออก
ตอนสกัดด้วย AI เนื้อหาไฟล์/เว็บ ถูกส่งไป LLM provider — BYO key ไม่ได้แปลว่า private
อย่าชี้ AI ไปที่ข้อมูลลับ
โฟลเดอร์ที่มี PII ลูกค้า/รหัสผ่าน → ใช้ keyword mode หรือ Ollama (local) แทน
Redaction เป็น best-effort
กรอง email/บัตร/key ด้วย regex แต่ ไม่การันตี 100% — อย่าพึ่งกับข้อมูลอ่อนไหวจริง
Flame Key = กุญแจทุกอย่าง
ใครมี key = decrypt + เข้า team ได้ทั้งหมด — เก็บเป็นความลับ ยังไม่มี per-member access
ตัวตนยังปลอมได้
ชื่อสมาชิกเป็น string ที่พิมพ์เอง — ยังไม่มี SSO ยืนยันตัวจริง (อยู่ใน roadmap)
ค้นหายังเป็น keyword
brain ใหญ่มากๆ (พันๆ facts) keyword อาจดึงไม่แม่น — semantic search เสียบเพิ่มได้ผ่าน interface
